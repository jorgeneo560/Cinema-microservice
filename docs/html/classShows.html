Detailed Description

Server-side movie show data structure with thread-safe seat management.

Represents a movie show with thread-safe seat management.

Identical to client-side Shows class but used in server context. Provides thread-safe operations for managing seat availability and bookings.

Thread Safety
All operations are thread-safe using shared_mutex for concurrent access.
See also
Shows in cinema_Client.hpp for detailed documentation

This class serves as the core data structure for the Cinema booking system. It provides thread-safe operations for managing seat availability and bookings for individual movie shows.

Thread Safety
All operations are thread-safe using shared_mutex for concurrent access. Multiple readers can access seat data simultaneously, but write operations are exclusive.
Seat Numbering
Seats are numbered 1-20 inclusive. Internal storage uses 0-based indexing.
Invariant
Total seats is always 20
Seat numbers are 1-20 inclusive
seats vector size is always 20
